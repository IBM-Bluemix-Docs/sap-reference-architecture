# 常见问题：IBM Cloud SAP 认证的基础架构的实施和运营

## IBM Cloud 为 SAP 提供受管服务器吗？
{: faq}

是的，IBM 提供 {{site.data.keyword.cloud}} for SAP Applications 产品的受管服务组件。有关更多信息，请参阅 [Managed Services for SAP Applications ![外部链接图标](../../icons/launch-glyph.svg "外部链接图标")](https://www.ibm.com/cloud/sap/managed){: new_window}。

## 如何在 IBM Cloud Infrastructure 上安装 SAP 应用程序？
{: faq}

SAP 应用程序是手动安装的，这是您的责任，因为没有预构建的应用程序映像。如果需要其他 SAP 受管服务，请联系 {{site.data.keyword.cloud_notm}} 服务销售专员。

## 在哪里可以找到有关如何在 IBM Cloud Infrastructure 上实施 SAP 的信息？
{: faq}

  * 有关如何在 {{site.data.keyword.cloud_notm}} 基础架构上实施 SAP NetWeaver 环境的信息，请参阅 [SAP NetWeaver on {{site.data.keyword.cloud_notm}}](/docs/infrastructure/sap-netweaver?topic=sap-netweaver-getting-started#getting-started)。

  * 有关如何在 {{site.data.keyword.cloud_notm}} 基础架构上实施 SAP HANA 环境的信息，请参阅 [SAP HANA on {{site.data.keyword.cloud_notm}}](/docs/infrastructure/sap-hana?topic=sap-hana-getting-started#getting-started)。

## 如何访问在 IBM Cloud 上运行的 SAP 系统？
{: faq}

初始，您可通过 {{site.data.keyword.cloud_notm}} 基础架构客户门户网站来访问您的基础架构。该客户门户网站提供额外的连接选项，包括 VPN。由于这是一个自我管理方案，因此实施所选的连接选项是您的责任。

## 如何将在 IBM Cloud Infrastructure 上运行的 SAP 系统连接到内部部署系统？
{: faq}

连接内部部署系统与 {{site.data.keyword.cloud_notm}} 基础架构是您的责任，因为这是一个自我管理产品。{{site.data.keyword.IBM_notm}} 还提供其他连接选项。有关更多信息，请参阅 [{{site.data.keyword.BluDirectLink}} ![外部链接图标](../../icons/launch-glyph.svg "外部链接图标")](https://www.ibm.com/cloud/direct-link){: new_window}。

## 如何对在 IBM Cloud Infrastructure 上运行的 SAP 系统执行备份？
{: faq}

自我管理产品中不包含备份。但是可以通过 {{site.data.keyword.cloud_notm}} 基础架构客户门户网站购买备份解决方案。您还可以选择购买和集成第三方备份解决方案。

## 如何将内部部署系统迁移到 IBM Cloud？
{: faq}

自我管理服务不随附任何迁移服务器；执行任何迁移活动都是您的责任。{{site.data.keyword.IBM_notm}} 提供一系列咨询和受管服务，可以帮助您计划和执行 SAP 迁移。有关更多信息，请参阅 [{{site.data.keyword.cloud_notm}} Advisory Services ![外部链接图标](../../icons/launch-glyph.svg "外部链接图标")](https://ibm.com/us-en/marketplace/cloud-consulting-services)。选择*服务* > *云服务*。
